# Background

The original website was created by someone else and later maintained through an administrator account for routine updates such as uploading photos, sermon videos, and service-related content.

The site was based on XE, a legacy CMS that became increasingly difficult to maintain over time. After hosting or platform environment changes, the previous site layout and maintenance workflow became unreliable.

The rebuild project started from the need to restore a stable public-facing website and create a structure that could be maintained more easily in the future.

## Original Situation

- Existing XE-based website
- Administrator access used mainly for photos and videos
- Limited ability to modify or modernize the structure
- Increasing maintenance risk due to legacy CMS limitations

## Problem

The previous site became difficult to maintain after CMS support limitations and hosting/platform changes affected the old structure.

## Rebuild Decision

The website was rebuilt using WordPress to provide a more maintainable structure for public information, media content, gallery pages, and future integrations.
